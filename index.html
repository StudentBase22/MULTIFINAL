01

Introduction to Multimedia Systems

What multimedia is, its history, human senses, applications, Flash basics, and design principles.

🎯

What is Multimedia?

Computer information represented through multiple media types — not just text.

  • Text, graphics, drawings, images
  • Audio, video, and animation
  • All data represented, stored, processed & transmitted digitally
Multi = many Media = communication means
📜

Vannevar Bush & the Memex (1945)

The visionary system that inspired hypertext, hyperlinks, and the World Wide Web.

  • Microphotography — entire encyclopedias in a matchbox
  • Voice-operated typewriters
  • Associative trails linking related documents
  • Desk with massive storage, projection & keyboard
👁️

Human Senses in Multimedia

Multimedia exploits multiple senses for richer experiences.

  • Sight — images, video, text, color
  • Hearing — speech, music, sounds
  • Touch — shape, texture, temperature
  • Smell & Taste — food, locations, objects
🏥

Applications of Multimedia

  • Entertainment — video-on-demand, interactive cinema, collaborative games
  • Healthcare — patient records, visual data sharing, PC conferencing
  • Education — lecture archives, intelligent textbooks, video conferencing
  • Business — training, marketing, simulations, catalogs
  • GIS — spatial data, zoomable maps with linked media
⚡

Flash Multimedia Basics

  • Timeline — frames, layers, keyframes control visibility & motion
  • Symbols — reusable assets with Motion Tween animation
  • ActionScript — play(), stop(), trace() for interactivity
  • Publishing — .fla source → .swf for web browsers
✅

Design Principles & When NOT to Use

Good multimedia should be clear, short (<5 min), user-centered with stop/forward/skip controls, and include metadata.

  • Don't use it just because you can
  • Don't use it only for the "Wow" factor
  • Every element must have a clear purpose
02

Media and Data Streams

System components, compression, media types, transmission modes, and streaming.

🖥️

System Components

  • Capture — cameras, mics, keyboards, VR devices
  • Storage — hard disks, CD/DVD-ROMs
  • Networks — LANs, intranets, internet
  • Processing — desktops, MPEG/DSP hardware
  • Display — speakers, HDTVs, monitors, printers
📊

Media Types & Storage

  • Graphics — lines, polygons; small & editable (Illustrator, Flash)
  • Images — 1-bit B&W, 8-bit grayscale, 24-bit true color (~0.75 MB for 512×512)
  • Audio — CD quality: 16-bit, 44.1 kHz (~5 MB/min mono)
  • Video — 25–50 fps; HD 1440×1080 ≈ 112.5 MB/min
Compression essential MP3 ZIP
⏱️

Discrete vs Continuous Media

  • Static (Discrete) — time-independent: text, graphics, still images
  • Continuous — time-dependent: audio, video, animation
  • Analog signals from the real world must be digitized for computers
📡

Transmission Modes

  • Asynchronous — fast as possible, arbitrary delays (text, chat)
  • Synchronous — continuous flow with buffering
  • Isochronous — steady flow for multimedia; reduces jitter (audio pops, video freezing)
🎬

Streaming Multimedia

Media stays on the server; playback begins before the full file arrives.

  • VCR-like controls: pause, rewind, fast-forward
  • Startup delay ~10 sec; command response 1–2 sec
  • Multiple quality versions for different network speeds
  • Adaptive streaming for modern systems
🔗

IPB Compression & Data Streams

  • I-frame, P-frame, B-frame — reduces size, maintains quality
  • Strongly regular — uncompressed video, CD audio
  • Weakly regular — compressed video with variable packets
  • Irregular — only frame changes transmitted

Real-time delay: audio <150 ms (good), <400 ms (acceptable).

03

Making Multimedia

Production stages, hardware/software requirements, creativity, and teamwork.

🔄

Four Stages of a Multimedia Project

1. Planning & Costing Develop idea, objectives, prototype, budget, timeline, navigation structure, and "look & feel."
2. Designing & Producing Create all multimedia elements. Client feedback may require many revision cycles.
3. Testing Verify objectives, navigation, and that all elements function properly.
4. Delivering Package final product — CD-ROM or web version, labels, and user manuals.
💻

Hardware & Software

  • Two major platforms: Macintosh & Windows
  • Mac preferred for sound/video editing
  • Tool set: authoring systems, text/image/sound/video editors
🎨

Essential Skills

  • Creativity — attractive, meaningful output
  • Organization — outlines, schedules, budgets
  • Teamwork — cooperation among specialists
Talent Skill Time Money

05

Text in Multimedia

Typography, fonts, design guidelines, hypertext, and font technologies.

🔤

Typography Essentials

  • Typeface — family (Arial, Times, Helvetica)
  • Font — one size + style (Bold, Italic)
  • Serif — decorative endings (Times)
  • Sans Serif — clean, no decorations (Arial)
  • 1 point = 1/72 inch; fonts below 12pt are hard to read
📐

Text Design Guidelines

  • Balance text amount — avoid too much or too little
  • Use few font faces (avoid "ransom-note typography")
  • Adjust leading (line spacing) & kerning (letter spacing)
  • Web pages: 1–2 screenfuls, minimize user actions
  • Short paragraphs — screen reading causes eye fatigue
🔗

Hypertext & Hypermedia

  • Hypertext — text linked to other information
  • Hypermedia — linked multimedia structure
  • Components: Nodes (content), Links (connections), Anchors (reference points)
  • Ted Nelson coined "hypertext" in 1965; Doug Engelbart invented the mouse
🛠️

Font Technologies

  • PostScript — Bezier curves, scalable, 6,000+ typefaces
  • TrueType — Apple & Microsoft, smooth & anti-aliased
  • Unicode — 65,000 characters, all languages & scripts
  • ASCII — most common character set
WYSIWYG Anti-aliasing Localization
🔍

Searching & Navigation

  • Boolean search: AND, OR, NOT
  • Truncation — partial word matching (e.g., "geo" → geology, geography)
  • Search engine robots visit pages and build indexes
  • Reading can be linear or non-linear
06

Sound in Multimedia

Sound physics, MIDI vs digital audio, file calculations, editing, and production standards.

🔊

Sound Physics

Vibrations → Pressure Waves → Perception by the ear and brain.

  • Amplitude — volume/loudness
  • Frequency — pitch (human range: 20 Hz – 20 kHz)
  • Harmonics — timbre (why a cello ≠ piano on same note)
0 dB — hearing threshold 60 dB — conversation 120 dB — pain threshold
🎹

MIDI

Musical Instrument Digital Interface — numeric commands, not recorded sound. Like a musical score.

  • Very small files (5–50 KB), loads instantly
  • Device-dependent — quality varies by synthesizer
  • Ideal for instrumental music, not speech
  • Requires: MIDI keyboard, synthesizer, sequencer software
🎙️

Digital Audio

Mathematical representation of analog sound waves as discrete samples.

  • Sampling Rate — 44.1 kHz = CD quality
  • Bit Depth — 16-bit standard; more bits = better quality, larger files
  • Mono — 1 channel; Stereo — 2 channels (L/R)
  • Device-independent — sounds identical everywhere
⚖️

MIDI vs Digital Audio

Feature MIDI Digital Audio
Data TypeMusical instructions/scoresRecorded sound samples
File Size5–50 KB (tiny)50–100 MB (large)
QualityDepends on playback deviceDepends on recording parameters
FlexibilityHighly editableFixed — editing risks integrity
Best ForInstrumental background tracksVoice, speech, acoustic instruments
🧮

File Size Formulas

Mono (Bytes) = Rate × Duration × (BitDepth / 8) × 1
Stereo (Bytes) = Rate × Duration × (BitDepth / 8) × 2

Example: 10s stereo at 44.1 kHz, 16-bit ≈ 1.68 MB

🎛️

Production & Standards

  • Three steps — Balance, Set Levels, Edit
  • Red Book Standard — CD: 44.1 kHz, 16-bit, 74 min
  • Lossless (FLAC) — 100% quality, archiving
  • Lossy (MP3) — smaller files, streaming/mobile
  • Vaughn's Law — "good enough" often beats over-engineering
⚠️

Integration & Copyright

  • 4-step workflow: Plan → Choose → Acquire → Edit & Test
  • Sync challenges: drift & latency across machines
  • Build organized sound libraries (FX, Voices, Ambience)
  • Always obtain legal permission for copyrighted work
  • Royalty-free clips — one-time purchase, multi-use
