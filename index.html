Introduction to Multimedia Systems

What multimedia is, its historical origin, human senses, applications, Flash basics, and design principles.

🎯

What is Multimedia?

Computer information represented using text, graphics, drawings, images, audio, video, and animation instead of traditional text-based information.

Multi = many, more than one

Media = means by which information is communicated

📜

Historical Origin - Vannevar Bush (1945)

Proposed the Memex system after WWII with these ideas:

  • Microphotography: Documents reduced in size (e.g., entire Encyclopedia Britannica in matchbox size)
  • Voice-Operated Typewriters: Speech-driven typing systems
  • Automatic Transaction Processing: Automatic card selection and data reading
  • Memex Desk: Desk-like machine with microfilm storage, projection surfaces, keyboard
  • Associative Trails: Links connecting related documents (inspired hypertext, hyperlinks, WWW)
👁️

Multimedia and Human Senses

Multimedia exploits multiple human senses:

  • Sight: Images, video, text, color
  • Hearing: Speech, music, sounds
  • Touch: Shape, texture, temperature
  • Smell: Food, objects, locations
  • Taste: Food, drinks, objects
🎮

Applications of Multimedia

  • Entertainment: Video-on-demand, interactive cinema, collaborative games, home shopping
  • Healthcare: Multimedia patient records, visual medical data sharing, PC conferencing
  • GIS: Spatial data storage, map zooming/panning, associating media with locations
  • Education: Lecture archives, video conferencing, intelligent multimedia textbooks
  • Manufacturing/Engineering: Team collaboration, enhanced CAD/CAM systems
  • Business: Animated ads, group communication tools, interactive training, simulations
🏫

Multimedia in Everyday Life

  • Schools: Educational software improves learning (e.g., "Just Grandma and Me")
  • Medical Education: Learn new technologies and procedures at own pace
  • Home: TV-based multimedia, family tree software, interactive entertainment
  • Public Places: Kiosks provide information, help services, public guidance
  • Business: Presentations, training, marketing, advertising, product demos, databases, catalogs
⚡

Flash Multimedia Basics

Main Components:

  • Toolbar (drawing tools)
  • Frames (time positions in animation)
  • Layers (organize objects)
  • Library (stores reusable assets)
  • Keyframes (change position/size/add objects)
  • ActionScript (programming language)

Publishing: .fla source → .swf output for web browsers

✅

User-Friendliness in Multimedia Design

Good multimedia should:

  • Have acceptable file size
  • Have clear media (audio, images)
  • Be short (less than ~5 minutes)
  • Show one item at a time
  • Use dialogue boxes for engagement
  • Be user-centered (stop, forward, skip, replay)
  • Include metadata (name, subject, keywords)
  • Show progress indicators
❌

When NOT to Use Multimedia

Do not use multimedia:

  • Just because you can
  • Only for the "Wow" factor
  • Without creativity

Key Rule: Every multimedia element must satisfy a clear identified purpose.

02

Media and Data Streams

Components of multimedia systems, compression, graphics, images, audio, video, and data stream characteristics.

🔧

Components of a Multimedia System

Capture Devices: Video cameras, microphones, keyboards, mice, graphics tablets, 3D input devices, VR devices

Storage Devices: Hard disks, CD-ROMs, DVD-ROMs

Communication Networks: LANs, Intranets, Internet, high-speed multimedia networks

Computer Systems: Multimedia desktops, workstations, MPEG hardware, video hardware, DSP hardware

Display Devices: Speakers, HDTVs, monitors, color printers

📦

Compression

Compression reduces file size for easier storage, faster transmission, and convenient archiving.

Benefits:

  • Easier storage
  • Faster transmission
  • Convenient archiving

Example: ZIP compression

�

Graphics

Characteristics: Constructed from lines, polygons, circles, curves, arcs

Creation: Graphics editors (Illustrator), Programs (Flash)

Advantages: Editable, revisable, small storage requirements

Input Devices: Keyboard, mouse, trackball, graphics tablet

🖼️

Images

Images obtained by scanners and digital cameras.

Storage Formats:

  • 1-bit: Black and White
  • 8-bit: Grayscale, Color map
  • 24-bit: True color

Storage Requirements:

  • 512×512 Grayscale (8-bit): ≈ 0.25 MB
  • 512×512 True Color (24-bit): ≈ 0.75 MB
  • Modern Cameras (10+ megapixels): ≈ 29 MB uncompressed
🎵

Audio

Audio signals are continuous analog signals captured by microphones and digitized.

CD Quality Audio: 16-bit sampling

Higher Quality: 24-bit

Storage Requirements:

  • Mono Audio (1 minute): ≈ 5 MB
  • Stereo Audio (1 minute): ≈ 10 MB

Compression: MP3

🎬

Video

Video captured by cameras and digitized, consisting of a sequence of images.

Typical Frame Rates: 25 fps, 30 fps, 50 fps

Storage Requirements:

  • HD DVD Video (1440×1080): ≈ 112.5 MB per minute

Note: Compression is essential for video.

📊

Discrete vs Continuous Media

Static (Discrete) Media: Time-independent (Text, Graphics, Still images)

Continuous Media: Time-dependent (Audio, Video, Animation)

📡

Analog and Digital Signals

Analog Signals: Continuous signals from real world (microphones, sensors, thermocouples, transducers)

Digital Signals: Discrete signals suitable for computers

Note: Analog signals must be digitized before processing.

⏱️

Time Dependency

Asynchronous Transmission: Data sent as fast as possible, arbitrary delays allowed (good for text/chat)

Synchronous Transmission: Continuous transmission, no start/stop bits, requires temporary buffering

Isochronous Transmission: Designed specifically for multimedia, maintains steady data flow, reduces jitter (audio pops, video freezing)

📐

IPB Compression

IPB stands for: I-frame, P-frame, B-frame

Purpose: Reduce file size while maintaining quality

Data Stream Types:

  • Strongly Regular: Uncompressed video, CD audio (consistent data rate)
  • Weakly Regular: Compressed video with large and small packets (moderate variation)
  • Irregular: Only changes between frames transmitted (highly variable rate)
🎞️

Logical Data Units (LDUs)

Continuous media represented as Logical Data Units (LDUs).

Examples: Audio samples, video frames

Important: Information defined by both its value AND the time it occurs.

📺

Streaming Stored Multimedia

Streaming means:

  • Media remains stored at server
  • Data transmitted continuously
  • Playback starts before whole file arrives

Features: VCR-like controls (pause, rewind, fast forward, slider navigation)

Acceptable Delays: Initial startup ≈ 10 seconds, Command response ≈ 1–2 seconds

Making Multimedia

Multimedia production stages, requirements, hardware, software, creativity, organization, and teamwork.

🎬

Introduction to Multimedia Production

Multimedia production creates presentations/applications combining text, graphics, audio, video, animation, and interactivity.

Requires: Hardware, Software, Creativity, Organization, Teamwork, Time and money

Good multimedia is about planning, skills, and cooperation among team members.

📋

Stages of a Multimedia Project

1. Planning and Costing: First and most important stage. Plan text, graphics, music, audio, video, develop layout and navigation, estimate time and cost, create prototype.

2. Designing and Producing: Perform planned tasks, create multimedia elements, develop final product. Client feedback may require repeated modifications.

3. Testing: Ensure project meets objectives, satisfies client needs, navigation works correctly, multimedia elements function properly.

4. Delivering: Package and deliver final product (CD-ROM or web version, labels, user manuals).

💻

Hardware Requirements

Important Platforms: Macintosh OS and Microsoft Windows

Macintosh Advantages: Better sound/video editing support, Apple hardware handles multimedia more efficiently

Both systems provide: GUI, affordability, available hardware and software

🛠️

Software Requirements

Software gives instructions to hardware.

Types of Multimedia Software:

  • Production tools
  • Authoring tools

Basic Multimedia Tool Set: Authoring systems, text editing, image editing, sound editing, motion video editing software

🎨

Creativity

Creativity means using knowledge of hardware and software effectively to produce attractive and meaningful multimedia.

A creative multimedia developer understands: Technology, Design, Communication

📊

Organization

Organization is essential. Developers must prepare:

  • Detailed outlines
  • Time schedules
  • Budgets
  • Resource lists
  • Skill requirements
👥

Teamwork in Multimedia

Multimedia production requires cooperation among specialists, shared responsibilities, and communication among team members.

Key Idea: "To make GOOD multimedia you need: Talent, Skill, Organization, Time, Money, Help from others"

Therefore: Teamwork is essential.

05

Sound

Sound physics, multimedia system sound, MIDI, digital audio, audio editing, production standards, and integration.

🔊

Introduction to Sound & Sound Physics

Sound is energy generated by vibrating molecules traveling through the air as pressure waves.

Progression: Vibrations → Pressure Waves → Perception (Ear detects/Brain interprets)

Key Characteristics:

  • Amplitude: Dictates volume/loudness
  • Frequency/Wavelength: Determines pitch
  • Harmonics: Shape distinct "timbre" (why cello ≠ piano on same note)
💻

Multimedia System Sound

Multimedia systems use sound to transform user experience from static to dynamic.

Macintosh: Built-in sounds (glass, indigo, laugh), stored as data files or resources

Windows: Windows/Media directory, startup/notification sounds (start.wav, chimes.wav, chord.wav)

Computer Standards: Built-in sound recorders, standard sound cards, audio options in software suites

🎹

MIDI (Musical Instrument Digital Interface)

Developed in 1980s as communications standard. Not digitized sound - music stored in numeric format (like musical score).

How it Works: Lists of mathematical commands sent to playback device

System Dependencies: Highly device-dependent (quality varies by synthesizer)

Use Case: Ideal for instrumental music, not suitable for spoken dialogue

Production Toolkit: Musical knowledge, MIDI keyboard, sound synthesizer, sequencer software

🎧

Digital Audio

Mathematical representation of analog sound waves as discrete samples.

Key Parameters:

  • Sampling Rate: Measured in kHz (11.025 kHz, 22.05 kHz, 44.1 kHz CD quality)
  • Sample Size (Bit Depth): 8-bit or 16-bit (more bits = better quality, larger files)
  • Quantization: Rounding sampled wave value to nearest integer

Stereo vs Mono: Mono (1 channel), Stereo (2 channels - L/R, more realistic but double storage)

Independence: Device-independent (sounds identical on all devices)

⚖️

MIDI vs Digital Audio Comparison

Feature MIDI Digital Audio
Data Type Musical instructions/scores Actual recorded sound
Storage Form Numeric commands Discrete samples
File Size Very small (5-50 KB) Large (50-100 MB)
Quality Basis Depends on playback device Depends on recording parameters
Flexibility Highly editable Fixed (editing risks integrity)
Device Reliance Device-dependent Device-independent
Best For Synthesized instrumental tracks Voice, speech, acoustic instruments
🎛️

Audio Editing & Production Standards

Three Steps: Balance (quality vs RAM/storage), Set Levels (adjust recording input), Edit (refine/polish)

Basic Operations: Trimming, Splicing & Assembly, Volume Adjustments, Multi-track Editing

Production Tools: QuickTime Player Pro, Sound Forge, Scripting Playback Engines

Red Book Standard (ISO 10149): 44.1 kHz, 16-bit linear, 74 minutes audio, CD-ROM/XA support

📦

Audio Compression Techniques

Lossless (e.g., FLAC): Reduces space while keeping 100% original quality. Best for archiving and high-end listening.

Lossy (e.g., MP3): Discards less noticeable details for smaller files. Best for streaming and mobile storage.

Vaughn's Law: "Good enough" configuration often beats over-engineering. Focus on meeting project requirements.

🔄

Integration & Workflow Challenges

4-Step Workflow: Plan → Choose (MIDI/Digital Audio/mix) → Acquire → Edit & Test

Delivery: Download (wait for whole file) vs Streaming (play while downloading)

Synchronization Challenges: Drift (audio/video out of sync), Latency (processing delays)
